# 박람회 리드 후속 이메일 자동화 (키즐링)

박람회에서 수집한 리드 명단 CSV를 읽어 **직책별(원장/팀장/사업개발) 소구점이 반영된 개인화 후속 이메일 초안**을 생성하는 4단계 파이프라인.
각 단계의 상세 규칙은 `.claude/skills/analyze.md`, `insight.md`, `generate.md`, `review.md`에 있으며, 이 파일은 전체 흐름을 하나로 묶은 진입점이다.

## 사전 준비
- [ ] `context/company-info.md`, `context/industry-news.md` 로 회사 맥락·직책별 소구점 정의 확인
- [ ] 입력 CSV 스키마 확인: `lead_id,name,organization,org_type,title,email,consultation_note,interest_feature,follow_up_priority`
- [ ] 기본 입력 파일: `data/leads_sample.csv` (새 리드 CSV로 교체해도 동일 파이프라인 적용 가능)

## 절차 체크리스트

### 1. 분석 (`.claude/skills/analyze.md`)
- [ ] CSV 전체 행 읽기, 컬럼 확인
- [ ] 직책(title) 분포 집계 (원장/팀장/사업개발 + 신규 직책값 여부)
- [ ] interest_feature·org_type·follow_up_priority 분포 집계
- [ ] email 결측 리드 목록화 (발송 불가 대상)
- [ ] consultation_note 결측·모호 리드 목록화 (review_flag=Y 후보)
- [ ] 완전 중복 리드(lead_id만 다르고 name·organization·note 동일) 탐지
- [ ] 분석 요약 출력

### 2. 소구점 전략 수립 (`.claude/skills/insight.md`)
- [ ] 직책별 소구점·제품 연결·CTA 확정
  - 원장: 원생 재능발굴·학부모만족·브랜딩 → 무료 데모 신청
  - 팀장: 도입난이도·운영공수절감·연동 → 기술 도입 미팅
  - 사업개발: 파트너십 수익모델·공동마케팅 → 파트너십 제안서 송부
- [ ] review_flag 기준 확정 (note 정보량 단일 기준: 공란/결측 표현/모호 문장 → Y)
- [ ] 이메일 없는 리드 예외 처리 규칙 확정 (후속 메모 + 대체 채널 + 우선순위별 기한)
- [ ] 중복 리드 처리 규칙 확정 (정본 1건만 초안 생성, 나머지 "중복 — 제외" 표기)
- [ ] follow_up_priority(상/중/하) × title CTA 강도 매트릭스 적용
- [ ] 신규/미확인 title 세그먼트 매핑 규칙 적용 (예: 교장→원장, PM→팀장, 주무관→사업개발; 그 외는 review_flag=Y)

### 3. 이메일 초안 생성 (`.claude/skills/generate.md`)
각 리드에 대해 본문 250~400자(한국어), 4요소 순서 고정으로 생성:
- [ ] ① 맞춤 인사 (이름 + 직책 호칭)
- [ ] ② 부스 상담내용 언급 (구체적 1문장 참조, review_flag=Y는 일반화 문구)
- [ ] ③ 직책 소구점 → 제품(키즐콘 기능) 연결
- [ ] ④ 명확한 CTA 1개 (직책×우선순위 매트릭스 그대로 적용, 2개 이상 금지)
- [ ] 제목(subject) 직책별 차별화
- [ ] 이메일 없는 리드 → 후속 액션 메모 행으로 대체
- [ ] `output/email_drafts.csv` (컬럼: `lead_id,subject,body,review_flag`) 저장

### 4. 품질 검수 (`.claude/skills/review.md`)
- [ ] 전체 리드 커버리지 확인 (중복은 "제외" 표기로 처리됐는지)
- [ ] 원장·팀장·사업개발 3직책 소구점·제목·CTA가 서로 다른지 비교 확인
- [ ] 개인화 4요소 누락 여부 전수 확인
- [ ] body 글자수(250~400자)·자연스러운 한국어·review_flag 정확성 확인
- [ ] 제출 형식(lead_id, subject, body, review_flag 컬럼) 확인
- [ ] 실패 항목 있으면 해당 이메일만 부분 재생성

### 5. 의사결정 로깅
- [ ] 직책별 소구점 매핑, review_flag 기준, 이메일 없는 리드 처리, 중복 리드 처리 등 중요 결정을 `decisions.md`에 기록
  (트리거 문구: "이 결정을 decisions.md에 기록해줘")

## 재현 실행 방법 (새 리드 CSV로 교체 시)
1. 새 CSV를 `data/` 아래에 두기 (스키마 동일해야 함)
2. 이 Skill을 다시 실행 → 위 1~5단계를 그 파일 기준으로 재수행
3. 원장/팀장/사업개발 외 title이 등장해도 `insight.md`의 신규 직책값 매핑 규칙에 따라 자동 분기됨
4. 결과를 원본 제출물과 구분해 별도 파일(예: `output/email_drafts_test.csv`)로 저장 가능

## 입출력 예시

### 예시 1 — 정상 케이스 (원장, 상담 메모 충분)
입력 (`data/leads_sample.csv` 1행):
```
L001,김민준,한빛영어학원,학원,원장,minjun.kim@hanbit-edu.co.kr,
"원생 180명 규모 영어학원. 매년 영어스피치 대회를 여는데 접수·심사·결과발표를 전부 수기로 해서 힘들다고 함. 원생 재능을 학부모에게 보여줄 수단을 찾고 있음.",
대회운영SaaS,상
```
출력 (`output/email_drafts.csv` 1행):
```
lead_id,subject,body,review_flag
L001,"[키즐링] 김민준 원장님, 매년 여는 영어스피치 대회를 자동화하는 방법 — 키즐콘 안내",
"김민준 원장님, 에듀플러스위크 부스에서 나눠주신 말씀 감사했습니다. 한빛영어학원에서 원생 180명 규모로 매년 영어스피치 대회를 여시는데 접수부터 심사·결과발표까지 수기로 진행하시느라 힘드시다는 이야기가 기억에 남습니다. 키즐콘은 대회 접수·결제·심사·결과발표를 한 번에 자동화하고 원생들의 발표 영상을 학부모님께 바로 공유해 재능을 보여주고 재등록률을 높이는 데 도움을 드립니다. 이번 주 안으로 무료 데모 일정을 잡아드리고 싶습니다. 편하신 시간 알려주시면 바로 안내드리겠습니다.",N
```

### 예시 2 — 예외 케이스 (이메일 결측 → 후속 메모로 대체)
입력: `L009,류현우,에듀빌영재교육원,학원,팀장,,...(email 컬럼 공란),콘텐츠안전필터,중`

출력 (`output/email_drafts.csv` 해당 행):
```
lead_id,subject,body,review_flag
L009,"[내부메모] 이메일 없음 — 후속 액션 필요","류현우 팀장 / 에듀빌영재교육원 / follow_up_priority=중. 이메일 정보 없어 발송 불가. 대체 채널(전화/방문/주최측 연락처 요청)로 72시간 내 후속 권장.",Y
```
(상담 메모가 결측·모호했다면 같은 방식으로 review_flag=Y, 본문은 부스 대화를 일반화한 문구로 대체)
