# /generate — 리드별 개인화 이메일 초안 생성

## 목적
/analyze + /insight 결과를 바탕으로 50건 리드 각각에 대한 개인화 이메일 초안을 생성하고 output/ 폴더에 저장한다.

## 전제 조건
- /analyze, /insight 완료 후 실행
- output/template.md 참고하여 형식 유지

## [Standard] 직책 정규화 (title → 세그먼트 매핑)
title이 원장/팀장/사업개발이 아니면 /insight의 "신규 직책값 세그먼트 매핑 규칙"에 따라 먼저 세그먼트를 확정한다.
```
교장 → 원장 세그먼트 (호칭: "{name} 교장님")
PM   → 팀장 세그먼트 (호칭: "{name}님")
주무관 → 사업개발 세그먼트 (호칭: "{name} 주무관님")
그 외 미확인 title → org_type·consultation_note로 가장 가까운 세그먼트 추정, 판단 애매하면 review_flag=Y
```
소구점·제품 연결·CTA는 매핑된 세그먼트 규칙을 그대로 따르고, 인사말 호칭만 원래 title 표기를 살린다.

## 이메일 생성 규칙 (개인화 4요소 필수)

각 이메일은 본문 **250~400자(한국어)** 로, 아래 4요소를 순서대로 포함:

```
① 맞춤 인사: "{name} {직책_호칭}" + 박람회(에듀플러스위크) 만남 감사
② 부스 상담내용 언급: consultation_note에서 나눈 구체적 대화 1문장 참조
③ 직책 소구점 → 제품 연결: title 소구점 + interest_feature(키즐콘 기능) 연결
④ 명확한 CTA 1개: 직책별 차별화 + follow_up_priority에 따라 강도 조절 (/insight의 "우선순위 × 직책 CTA 강도 매트릭스" 그대로 적용 — 직접 우선순위를 재산정하지 말 것)
```

### 제목(subject) — 직책별 차별화
```
원장:    [키즐링] {name} 원장님, 우리 원 아이들의 재능을 보여주는 방법 — 키즐콘 안내
팀장:    [키즐링] {name} 팀장님, 기존 시스템에 쉽게 붙는 대회 운영 자동화 — 키즐콘
사업개발: [키즐링] {name} 님, 키즐콘 공동 공급/파트너십 제안의 건
```

### 직책별 본문 톤 (예시 문장)
```
원장용:
  "{organization}처럼 원생의 재능을 학부모에게 보여주고 싶은 기관에서는,
  키즐콘으로 교내 챌린지·대회를 영상으로 기록·공유해 학부모 만족과 재등록을 높이고 계십니다."

팀장용:
  "키즐콘은 기존 운영 워크플로/시스템과 쉽게 연동되어, 접수·결제·심사·결과발표를
  자동화함으로써 대회 운영 공수를 크게 줄여드립니다."

사업개발용:
  "{organization}과 키즐링이 공동 공급/공동 마케팅으로 함께 성장하는 파트너십을 그려볼 수 있습니다.
  수익 배분 모델을 포함한 제안서를 정리해 보내드리겠습니다."
```

### review_flag = Y 리드 (상담 메모 결측·모호)
```
- body에서 상담 내용을 특정하지 않고 일반화:
  "에듀플러스위크 부스에서 잠깐 인사 나눴던 것 기억하실지 모르겠습니다. 바쁜 현장이라 짧게 뵈었지만,
   관심 보여주신 {interest_feature} 관련해 이렇게 연락드립니다."
- review_flag 컬럼에 Y 기록 → 발송 전 담당자가 상담 내용 보강
```

### 이메일 없는 리드 (email 결측)
```
이메일 초안 대신 후속 메모를 별도 작성:
=== 이메일 발송 불가 — 후속 액션 메모 ===
lead_id: {lead_id}
{name} / {organization} / {title}
follow_up_priority: {follow_up_priority}
사유: 이메일 정보 없음
권장 액션: 대체 채널(전화/방문/주최측 연락처 요청)로 후속, 기한 = 우선순위 기준(상=48시간 내)
```

### 중복 리드 (내용 완전 동일)
```
lead_id만 다른 중복 리드는 한 건만 초안 생성, 나머지는 비고에 "중복 — 제외" 표기
```

## 출력 파일 저장

### 옵션 A: CSV (권장 — 노션 ⑤ 명시 형식)
`output/email_drafts.csv`에 저장:
```
lead_id,subject,body,review_flag
```
- subject: 직책별 차별화 제목
- body: 4요소 포함 250~400자 한국어 (쉼표/줄바꿈 포함 시 큰따옴표로 감쌈)
- review_flag: Y(상담 메모 결측·모호) / N

### 옵션 B: XLSX
같은 컬럼으로 `output/email_drafts.xlsx` 저장 (셀 줄바꿈 허용으로 가독성↑)

## 완료 기준
- [ ] 발송 가능 리드: 이메일 초안 생성 (중복 1건 제외)
- [ ] 이메일 없는 리드: 후속 메모 생성
- [ ] review_flag 정확히 표기 (상담 메모 결측·모호 = Y)
- [ ] 원장/팀장/사업개발 소구점·CTA·제목이 서로 다름
- [ ] body 250~400자, 자연스러운 한국어
- [ ] output/ 폴더에 email_drafts.csv 또는 .xlsx 저장
