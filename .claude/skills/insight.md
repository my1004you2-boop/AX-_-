# /insight — 직책별 소구점 전략 수립

## 목적
/analyze 결과를 바탕으로 직책별 이메일 소구점 전략과 예외 처리(review_flag) 기준을 수립한다.

## 전제 조건
- /analyze 결과를 먼저 확인했을 것
- context/company-info.md의 "직책별 소구점" 섹션을 참고

## 직책별 소구점 매핑

### 원장·부원장 (학원장·기관장·원감)
```
핵심 소구점: 원생 재능발굴, 학부모 만족, 기관 브랜딩
제품 연결: interest_feature(대회운영SaaS / 콘텐츠안전필터)를 "원생 효과"로 연결
  - 대회운영SaaS → "교내 대회 접수·심사·결과발표를 자동화해 원생 성취를 보여준다"
  - 콘텐츠안전필터 → "안전하게 아이들 챌린지를 기록·공유해 학부모 만족과 재등록을 높인다"
이메일 톤: 비즈니스 오너 공감 + 원생/학부모 효과 중심
CTA: 무료 데모 신청 (운영 진단 / 원생 효과 체험)
```

### 팀장 (마케팅팀장·운영팀장·콘텐츠팀장)
```
핵심 소구점: 키즐콘 도입 난이도, 운영 공수 절감, 기존 시스템 연동
제품 연결: "기존 워크플로/시스템과 쉽게 연동, 대회·콘텐츠 운영 공수 절감"
이메일 톤: 실무 중심, 도입 난이도·연동·공수 절감 수치 강조
CTA: 기술 도입 미팅 (연동·도입 일정 협의)
```

### 사업개발 (BD·파트너십·제휴)
```
핵심 소구점: 파트너십 수익 모델, 공동 마케팅, 기관 공급 계약
제품 연결: "공동 공급·공동 마케팅·기관 공급으로 상호 수익을 만든다"
이메일 톤: 전략적·비즈니스적, 수익 배분·시장 기회 중심
CTA: 파트너십 제안서 송부 (사업개발 미팅 제안 포함)
```

## review_flag 기준
```
review_flag = Y (사람이 상담 내용을 보강한 뒤 발송):
  - consultation_note가 비어 있음 (빈 셀)
  - consultation_note = "상담 메모 없음" 등 결측 표현
  - 니즈 파악이 안 되는 모호한 메모

review_flag = N: 상담 내용이 충분히 구체적인 리드
```
→ Y인 리드는 본문에서 상담 내용을 일반화("부스에서 인사 나눴던 것 기억하실지…")하고, 발송 전 사람이 보강하도록 표시.

## 이메일 없는 리드 예외 처리
```
email이 비어 있는 리드: 이메일 초안 대신 후속 메모 생성
  - 발송 불가 사유 명시
  - 대체 채널(전화/방문/주최측 연락처 요청 등) 권장
  - follow_up_priority에 따라 후속 기한 제안 (상=48시간 내)
```

## 중복 리드 처리
```
lead_id만 다르고 내용 동일한 중복 리드: 한 건으로 처리하고 나머지는 "중복 — 제외" 표기
```

## [Standard] 우선순위 × 직책 CTA 강도 매트릭스
follow_up_priority(상/중/하)와 title(원장/팀장/사업개발) 9칸을 아래처럼 구체적으로 확정해 재현 파이프라인에서 항상 동일하게 적용한다.

| follow_up_priority | 원장 | 팀장 | 사업개발 |
|---|---|---|---|
| 상 (48시간 내) | 이번 주 무료 데모 신청 제안 | 이번 주 기술 도입 미팅 제안 | 제안서 송부 + 이번 주 미팅 제안 |
| 중 (72시간 내) | 무료 데모 안내 + 사례집 첨부 | 도입 효과 자료 공유 후 미팅 제안 | 파트너십 제안서 송부 |
| 하 (장기 너처링) | 소개 자료·사례집만 발송 | 소개서·연동 가이드 발송 | 파트너십 소개 자료만 발송 |

설계 근거: "하" 우선순위 리드에게 즉시 미팅·데모를 요청하면 부담을 줘 이탈시킬 수 있어 우선순위가 낮을수록 요청 강도를 낮추고 정보 제공 중심으로 전환한다. "상" 우선순위는 골든타임(48시간) 안에 붙잡아야 하므로 구체적 기한과 행동 요청을 명시한다.

> follow_up_priority는 주어진 입력값으로만 사용해 CTA 강도를 조절한다. 직접 점수를 매기지 말 것(스코어링 제외 범위). review_flag=Y 리드는 강도와 무관하게 사람 검토 후 발송.

## [Standard] 신규 직책값 세그먼트 매핑 규칙
새 리드 CSV에 원장/팀장/사업개발 외의 title이 등장하면 아래 고정 매핑을 따른다(interest_feature와는 무관하게 title 기준으로만 판단).

| 등장 title | org_type 예시 | 매핑 세그먼트 | 호칭 |
|---|---|---|---|
| 교장 | 학교 | 원장 (소구점·CTA 동일) | "OOO 교장님" |
| PM | 에듀테크기업 | 팀장 (소구점·CTA 동일) | "OOO님" (직함 존칭 없이) |
| 주무관 | 공공기관 | 사업개발 (소구점·CTA 동일) | "OOO 주무관님" |

매핑 근거: 교장은 학교의 기관장으로 원장과 결정 권한·관심사(원생 효과·학부모 만족·기관 브랜딩)가 동일하다. PM은 에듀테크기업의 실무 도입 검토자로 팀장과 역할이 같다(도입난이도·연동·공수 절감). 주무관은 공공기관에서 파트너십·기관 공급 계약을 실무선에서 논의하는 창구로 사업개발과 역할이 유사하다. 위 3개 외에 또 다른 미확인 title이 등장하면 org_type과 consultation_note를 참고해 가장 가까운 세그먼트로 매핑하고, 판단이 애매하면 review_flag=Y로 사람에게 넘긴다.
