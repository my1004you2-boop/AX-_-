# /review — 이메일 초안 품질 검수

## 목적

/generate로 생성한 이메일 초안이 채점 기준을 충족하는지 체크리스트로 확인한다.

## 전제 조건

- output/email_drafts.csv 또는 .xlsx 생성 완료

## 검수 체크리스트

### A. 전체 커버리지

- [ ] 전체 50건 리드 모두 처리되었는가? (중복 1건은 "제외" 표기)
- [ ] 이메일 있는 리드: 초안이 존재하는가?
- [ ] 이메일 없는 리드: 후속 메모가 생성되었는가?
- [ ] 상담 메모 결측·모호 리드: review_flag = Y로 표기되었는가?

### B. 직책별 소구점 차별화 (핵심, 40점)

원장·팀장·사업개발 이메일을 각 1개씩 꺼내어 비교:

| 항목        | 원장                  | 팀장                  | 사업개발              |
| ----------- | --------------------- | --------------------- | --------------------- |
| 핵심 소구점 | 원생효과·학부모만족? | 도입난이도·공수절감? | 수익모델·파트너십?   |
| 제목 차별화 | 원생/재능 중심?       | 연동/운영 중심?       | 파트너십 중심?        |
| CTA         | 무료 데모 신청?       | 기술 도입 미팅?       | 파트너십 제안서 송부? |

- [ ] 3가지 직책 이메일이 서로 다른 소구점·CTA·제목을 쓰는가?

### C. 개인화 4요소 포함 여부 (20점)

각 이메일에 대해:

- [ ] ① 맞춤 인사 (이름 + 직책 호칭, 예: "김민준 원장님")
- [ ] ② 부스 상담내용 언급 (review_flag=Y 리드는 일반화 인정)
- [ ] ③ 직책 소구점 → 제품(키즐콘 기능) 연결
- [ ] ④ 명확한 CTA 1개

### D. 본문 품질·예외 처리 (30점)

- [ ] body가 250~400자 범위인가?
- [ ] 자연스러운 비즈니스 한국어인가?
- [ ] review_flag가 정확한가? (상담 메모 비었거나 모호 = Y, 그 외 N)
- [ ] 이메일 없는 리드가 후속 메모로 예외 처리되었는가?
- [ ] CTA가 우선순위(상/중/하)에 따라 강도가 조절되었는가? (/insight 매트릭스와 일치하는지)
- [ ] CTA가 2개 이상인 이메일은 없는가? (분산 방지)
- [ ] 내부 메모성 표현이 본문에 노출되지 않았는가?

### D-1. [Standard] 신규/미확인 직책값 매핑 (재현성 검증)

- [ ] 원장/팀장/사업개발 외 title(예: 교장, PM, 주무관 등)이 /insight 매핑 규칙대로 세그먼트·호칭이 적용되었는가?
- [ ] 규칙에 없는 완전히 새로운 title이 나오면 review_flag=Y로 넘겼는가? (임의로 소구점을 지어내지 않았는가)

### E. 제출 형식 (10점)

- [ ] 컬럼이 lead_id, subject, body, review_flag 인가?
- [ ] CSV/XLSX로 저장되었는가?



- [ ] 노션/구글 시트 공개 링크로 제출 준비되었는가?

### F. decisions.md 작성 확인

- [ ] 직책 소구점 매핑 근거가 기록되었는가?
- [ ] 이메일 없는 리드 처리 방법이 기록되었는가?
- [ ] review_flag 기준이 기록되었는가?
- [ ] 중복 리드 처리 방법이 기록되었는가?

## 검수 결과 출력 형식

```
=== 검수 결과 ===
총 처리: 50/50건 (중복 1건 제외 → 실질 49건)
이메일 초안: N건 / 후속 메모: N건 / review_flag=Y: N건

[통과/실패] A. 전체 커버리지
[통과/실패] B. 직책별 소구점 차별화
[통과/실패] C. 개인화 4요소
[통과/실패] D. 본문 품질·예외 처리
[통과/실패] E. 제출 형식
[통과/실패] F. decisions.md

개선 필요 항목:
1. ...
```

## 개선이 필요한 경우

검수에서 실패한 항목이 있으면 /generate를 해당 이메일만 부분 재실행.

## 완료 선언 기준

A~F 항목 모두 통과 시 "검수 완료" 선언.
