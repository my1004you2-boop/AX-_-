# [05] 키즐링 — 박람회 리드 맞춤 후속 이메일 자동화

당신은 키즐링의 콘텐츠 마케터입니다.
이 회사의 맥락과 문제를 충분히 이해한 후 미션을 수행하세요.

## 기업 컨텍스트
→ context/company-info.md 와 context/industry-news.md 를 먼저 읽으세요.

## 미션 요약
박람회에서 수집한 리드 명단 CSV를 분석하여, **직책별 소구점이 반영된 개인화 후속 이메일 초안**을 자동 생성하세요.
레벨(Basic → Standard → Challenge)은 본인 수준에 맞게 선택합니다. 위 단계는 아래 단계를 포함하는 누적형입니다.

## 권장 접근 순서
1. /analyze  → data/leads_sample.csv 파악: 직책 분포, 이메일 결측·상담 메모 결측/모호, 관심제품·우선순위 분포 확인
2. /insight  → 직책별 소구점 전략 설계(원장=원생효과·학부모만족 / 팀장=도입난이도·공수절감 / 사업개발=수익모델·파트너십) + review_flag 기준 수립
3. /generate → output/template.md 형식으로 리드별 이메일 초안(제목·본문·review_flag) 생성 → email_drafts.csv/.xlsx 저장
4. /review   → 제출 전 자가 점검 (직책 차별화·개인화 4요소·CTA·예외 처리)

## 의사결정 로깅 규칙
직책별 소구점 매핑, 이메일 없는 리드 처리, 상담 메모 결측/모호 시 review_flag 처리, 중복 리드 처리 등 중요 결정은
반드시 decisions.md 에 기록하세요.
→ "이 결정을 decisions.md에 기록해줘" 입력 시 자동 기록됩니다.

## 채점 기준 (누적형)

### 🟢 Basic — 100점 만점 (필수)
- 직책별 차별화 정확도 (40점): 원장·팀장·사업개발 3직책이 서로 다른 소구점·CTA를 쓰는가 + 제목이 직책별로 차별화
- 개인화 완성도 (20점): 각 이메일에 ①맞춤 인사 ②부스 상담내용 언급 ③직책 소구점→제품(키즐콘 기능) 연결 ④명확한 CTA 4요소 포함
- 본문 품질·예외 처리 (30점): 본문 250~400자 자연스러운 한국어 + review_flag 정확(상담 메모 비었거나 모호 시 Y) + 이메일 없는 리드 예외 처리
- 제출 형식 (10점): email_drafts.csv 또는 .xlsx (lead_id, subject, body, review_flag) — 노션/구글 시트 공개 링크로 제출

### 🟡 Standard 가산 — +30점
- 파이프라인 재현성 (15점): 새 리드 CSV를 넣어도 정제→직책 분기→초안 생성→review_flag 동일 품질로 자동 생성 (/명령 또는 스크립트)
- 후속 운영 설계 (15점): 직책별 소구점·CTA·우선순위별 강도를 본인이 직접 설계 + 마케팅·영업 관점 근거 설명

### 🔴 Challenge 가산 — +30점
- 후속 전략 실효성 (20점): 리드 데이터에서 발견한 패턴 기반 후속 캠페인 기획안 1~2건 (세그먼트 정의·근거 리드·후속 시퀀스·우선순위)
- 양식·기준 설계 (10점): 이메일 양식·review_flag 기준·우선순위 판단 기준을 본인이 설계 + 근거 설명

> 최소 합격선: Basic 60점 이상 (직책별 차별화 이메일 + 개인화 4요소 완성)
> 레벨별 만점: Basic 100 / Standard 130 / Challenge 160

## 제약 조건
- Claude Code (터미널) 또는 Cowork (채팅) 만 사용
- 별도 API 키 없음 — Claude Code 대화 안에서 직접 초안 생성 완료
- **이메일 실제 발송 제외** (SMTP·Gmail API 연동 금지) — 발송은 사람이 검토 후 진행, 과제 범위는 **초안 생성까지**
- **CRM 연동 제외** (Salesforce·HubSpot 등)
- 다국어 이메일 제외 (한국어만)
- 리드 스코어링 제외 — follow_up_priority(상/중/하)는 주어진 입력값으로만 사용 (직접 점수 산출 금지)
- 대시보드·웹 UI 구현 제외 (제공된 leads_sample.csv만 사용)
- Cowork 사용 시: 배포 URL 대신 Skill(/명령) 패키징으로 제출 가능
