## 2. 레벨 구조 설계

### Q. 현재 목표를 레벨 1로 하고 레벨 2, 3을 설정한다면?

**설계 원칙**: "1회성 초안" → "매 박람회마다 돌아가는 자동화" → "데이터 기반 후속 전략"으로 발전하는 누적형 구조.
콘텐츠 생성형 트랙 페르소나 사다리: **콘텐츠 마케터 → 마케팅 PM → 콘텐츠 전략가**.

### 최종 확정 레벨

```
🟢 Basic (100점)  — 콘텐츠 마케터 수준
  → 리드 50건을 직책별(원장/팀장/사업개발) 소구점이 다른 개인화 이메일 초안으로 생성
  → 개인화 4요소(인사·상담언급·제품연결·CTA) + review_flag + 이메일 없는 리드 예외 처리
  → 제출: email_drafts.csv/.xlsx + 노션/구글 시트 공개 링크

🟡 Standard (+30점, 누적 130점)  — 마케팅 PM 수준
  → Basic + 새 리드 CSV에도 동일 품질로 도는 재현 파이프라인 (/명령 또는 스크립트)
  → 직책별 소구점·CTA·우선순위별 강도를 본인이 직접 설계

🔴 Challenge (+30점, 누적 160점)  — 콘텐츠 전략가 수준
  → Standard + 리드 패턴 기반 후속 캠페인 기획안 (세그먼트·근거 리드·후속 시퀀스·우선순위)
  → 이메일 양식·review_flag 기준·우선순위 판단 기준도 본인이 설계
```

## 3. CSV 데이터 재설계

### Q. 기존 problem-05-kizzling 폴더와 노션 v2(Problem 05) 정합성 맞추기

**변경 전 컬럼** (구버전 — 가공의 회사·리드 관리 SaaS 설정)
```
id, 이름, 소속기관, 직책, 이메일, 상담내용요약, 관심제품, 후속우선순위, 담당자메모
```
- 17행, 직책에 "미확인/빈칸" 존재, 관심제품이 가공의 플랜명(리드 자동화 플랜 등)

**변경 후 컬럼** (노션 v2 기준 — 실제 키즐링·키즐콘)
```
lead_id, name, organization, org_type, title, email, consultation_note, interest_feature, follow_up_priority
```

**주요 변경 사항**
- 기업 설정 전면 교체: 가공의 "리드 관리 SaaS" → 실제 **키즐링(주)**(알파세대 안전 숏폼 챌린지 + LXP), B2B 제품 **키즐콘(Kizlcon)**(대회 원스톱 SaaS)
- `id` → `lead_id` (L001~L050 형식)
- `소속기관` → `organization`, `직책` → `title`, `이메일` → `email`
- `org_type` 신규 추가: 학원 / 학교 / 에듀테크기업 / 공공기관
- `관심제품`(가공 플랜명) → `interest_feature`: **대회운영SaaS / 콘텐츠안전필터 / 파트너십** (키즐콘 실제 기능군)
- `상담내용요약` → `consultation_note` (키즐콘 도메인 현실 메모로 전면 재작성)
- `후속우선순위`(높음/중간/낮음) → `follow_up_priority`(상/중/하) — **입력값으로만 사용, 직접 스코어링 금지**
- `담당자메모` 제거 (정답·내부 메모 노출 방지, 문제 스코프 외)
- `title`에서 "미확인/빈칸" 제거 — 노션 스펙이 3직책(원장/팀장/사업개발)만 명시
- **17행 → 50행 확장**, 구성 = 원장 20 / 팀장 18 / 사업개발 12

**정답 컬럼 금지**: 직책별 소구점·CTA·이메일 본문 등 참여자가 직접 산출해야 하는 결과는 데이터에 넣지 않음. `interest_feature`·`follow_up_priority`는 노션 ④/⑤가 입력값으로 명시한 hint성 컬럼이라 유지.

**의도적 데이터 품질 이슈 (참여자가 처리해야 할 과제)**
- 이메일 결측 1건: `L009` (이메일 없는 리드 → 발송 불가, 별도 후속 메모 전환 연습)
- 상담 메모 결측 2건: `L012`(="상담 메모 없음"), `L015`(빈 셀) → `review_flag = Y`
- 상담 메모 모호 1건: `L033`("관심은 있어 보이나 구체적 니즈 파악 안 됨") → `review_flag = Y`
- 완전 중복 리드 1건: `L050` = `L039` (lead_id만 다르고 name·organization·consultation_note 등 내용 완전 동일) → 한 건으로 처리

> review_flag 연습 포인트: consultation_note가 비었거나 모호한 L012·L015·L033은 Y, 그 외는 N. L046(="상담 메모 없음")도 결측이므로 Y로 처리되도록 추가 배치(사업개발). 즉 결측/모호 케이스가 직책별로 골고루 분포하도록 심음.
