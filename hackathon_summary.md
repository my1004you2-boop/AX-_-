# AX 해커톤 — Problem 05 "키즐링" 작업 요약

> 발표 자료 제작용 정리 문서. 작성일 2026-07-01.
> 이 프로젝트는 **①문제(Problem) 설계**와 **②그 문제의 예시 해결(참조 솔루션) 실행**을 모두 포함합니다.

---

## 1. 한 줄 요약

박람회에서 수집한 리드 CSV를 넣으면, **직책별(원장/팀장/사업개발) 소구점이 반영된 개인화 후속 이메일 초안**을 자동 생성하는 Claude Code 파이프라인을 설계하고, Basic → Standard → Challenge 3단계로 직접 실행해 참조 결과물까지 만들었다.

---

## 2. 기업 & 문제 배경

**키즐링(주)** — 알파세대(2010년대 이후 출생) 어린이 대상 **안전 숏폼 챌린지 플랫폼 + LXP** 운영 에듀테크 스타트업.
- 유해 콘텐츠 차단율 99%, CES 2026 혁신상 / 구글 창구 7기 / 중기부 디딤돌 R&D 선정
- B2B 핵심 제품: **키즐콘(Kizlcon)** — 학교·학원 대상 대회 접수·결제·심사·결과발표 원스톱 SaaS. 기능군 3가지: 대회운영SaaS / 콘텐츠안전필터 / 파트너십

**현업 문제(고통점)**
1. 리드 1명당 후속 이메일 작성 10~15분 → 50명이면 8시간 이상, 담당자 1인 감당 불가
2. 직책별 맥락 분기 미흡 (원장="원생 효과" / 팀장="도입 난이도" / 사업개발="수익 모델") — 같은 톤 이메일은 아무에게도 안 꽂힘
3. 상담 내용이 이메일에 반영되지 않아 "우리를 기억 못 하는 회사" 인상
4. 48시간 골든타임 이탈 — 늦으면 경쟁사에 리드를 빼앗김

## 3. 레벨 구조 (누적형, 페르소나 사다리)

| 레벨 | 페르소나 | 목표 | 배점 |
|---|---|---|---|
| 🟢 Basic | 콘텐츠 마케터 | 직책별 개인화 이메일 초안 50건 + 예외 처리(review_flag) | 100점 |
| 🟡 Standard | 마케팅 PM | 새 CSV에도 동일 품질로 도는 재현 파이프라인 + 본인 설계 CTA 강도 | +30점 (누적 130) |
| 🔴 Challenge | 콘텐츠 전략가 | 리드 패턴 기반 후속 캠페인 기획안 + 본인 설계 기준 | +30점 (누적 160) |

최소 합격선: Basic 60점 (직책별 차별화 이메일 + 개인화 4요소 완성)

**제외 범위**: 실제 이메일 발송(SMTP/Gmail API), CRM 연동, 다국어, 리드 스코어링(직접 산출 금지, 입력값만 사용), 대시보드/웹 UI.

## 4. 입력 데이터 설계 (`data/leads_sample.csv`)

- 50행: 원장 20 / 팀장 18 / 사업개발 12
- 컬럼: `lead_id, name, organization, org_type, title, email, consultation_note, interest_feature, follow_up_priority`
- `interest_feature`(대회운영SaaS/콘텐츠안전필터/파트너십), `follow_up_priority`(상/중/하)는 **입력 힌트값** — 직접 스코어링 금지

**의도적으로 심어둔 데이터 품질 이슈** (참여자가 처리해야 할 과제)
- 이메일 결측 1건 (L009)
- 상담 메모 결측 2건 (L012, L015) + 모호 1건 (L033), 추가 결측 1건(L046) → 총 review_flag=Y 후보
- 완전 중복 리드 1건 (L050 = L039)

재현성 검증용으로 100행짜리 별도 테스트 CSV(`data/leads_dummy_100_test.csv`)도 함께 제작.

## 5. 파이프라인 설계 (Claude Code Skill 4단계)

`.claude/skills/` 아래 슬래시 커맨드로 패키징:

| 커맨드 | 역할 |
|---|---|
| `/analyze` | 직책·기관유형·우선순위 분포, 이메일/상담메모 결측·모호, 완전 중복 탐지 |
| `/insight` | 직책별 소구점→제품 연결, CTA, review_flag 기준, 예외 처리 규칙 수립 |
| `/generate` | 리드별 이메일 초안(제목·본문·review_flag) 생성 → `output/email_drafts.csv` |
| `/review` | 직책 차별화·개인화 4요소·본문 품질·예외 처리 자가 점검 |

전체 흐름은 `.claude/skills/SKILL.md`에 하나의 Skill(`kizzling-lead-followup-email`)로 통합. 새 CSV로 교체 후 동일 Skill 재실행만으로 파이프라인 재현 가능.

## 6. 직책별 소구점·CTA 설계 (핵심 차별화 로직)

| 직책 | 핵심 관심사 | 소구 메시지 | CTA |
|---|---|---|---|
| 원장 | 원생 재능 발굴, 학부모 만족, 기관 브랜딩 | "재능을 영상으로 기록·공유해 학부모 만족·재등록을 높인다" | 무료 데모 신청 |
| 팀장 | 도입 난이도, 운영 공수 절감, 시스템 연동 | "기존 워크플로에 쉽게 연동, 운영 공수 크게 절감" | 기술 도입 미팅 |
| 사업개발 | 파트너십 수익 모델, 공동 마케팅, 기관 공급 계약 | "공동 공급·공동 마케팅으로 상호 수익을 만든다" | 파트너십 제안서 송부 |

**개인화 4요소** (전 이메일 필수): ①맞춤 인사 ②부스 상담내용 언급(1문장) ③직책 소구점→제품 연결 ④단일 CTA(2개 이상 금지). 본문 250~400자, 자연스러운 한국어.

**[Standard] 우선순위×직책 CTA 강도 매트릭스** (9칸, 본인 설계):
- 상(48h 내) → 즉시 행동 요청 (데모/미팅 즉시 제안)
- 중(72h 내) → 자료 제공 + 미팅 제안
- 하(장기 너처링) → 소개 자료만 발송, 압박 없음

## 7. 예외 처리 규칙 (review_flag / 결측 / 중복)

| 케이스 | 처리 방식 | 근거 |
|---|---|---|
| 상담 메모 공란·"메모 없음"·모호 | `review_flag = Y`, 본문은 일반화 문구로 대체 | 무리하게 지어내면 신뢰 훼손 — human-in-the-loop 표준 |
| 이메일 결측 (L009) | `subject=[내부메모]`, 본문에 대체 채널(전화/방문) 후속 권장 문구, `review_flag=Y` | 이메일 없는 리드에 발송 시도 자체가 금지 규칙 |
| 완전 중복 (L039=L050) | 정본 1건만 이메일 생성, 나머지는 "[중복 제외]" 표기 | 동일 인물 이중 발송 방지 |
| title="원장"이지만 실제 호칭 관례가 다른 기관 | 세그먼트(소구점·CTA)는 유지, 인사말 호칭만 교체 (교장님/센터장님/관장님) | 한국어 비즈니스 관례상 부적절한 호칭은 개인화 신뢰도 저하 |
| [Standard] 신규 직책값 (교장/PM/주무관) | 교장→원장, PM→팀장, 주무관→사업개발로 title 단일 기준 매핑 | 결정 권한·관심사가 유사한 세그먼트로 고정, interest_feature까지 보정하면 규칙 복잡화·재현성 저하 |

전체 결정과 이유·대안은 [decisions.md](decisions.md)에 8건 기록.

## 9. [Standard] 재현성 검증 결과

같은 규칙(직책 매핑·강도 매트릭스·review_flag 기준)을 코드/설계 변경 없이 100건 신규 CSV에 그대로 적용.

| 지표 | Basic (50건) | 재현성 테스트 (100건) |
|---|---|---|
| 이메일 결측 | 1건 (L009) | 1건 (L016) |
| 상담메모 결측·모호 | 4건 | 9건 |
| review_flag=Y 총계 | 5건 | 10건 |
| 완전 중복 | 1건 | 0건 |
| 신규 직책값 | 없음 | 교장 6 / PM 11 / 주무관 6 → 매핑 규칙대로 정상 편입 |

→ 결과는 `output/email_drafts_test.csv`로 Basic 제출물과 분리 보관하여 "새 CSV를 넣어도 동일 품질" 재현성을 입증.

## 10. [Challenge] 후속 캠페인 기획안 (2건)

리드 데이터 패턴 기반, 본인 설계 우선순위 기준(세그먼트 규모×의향 40% / 계약 임팩트 40% / 골든타임 민감도 20%)으로 선정.

**P0 — 에듀테크 팀장 연동·도입 데모 트랙**
- 세그먼트: `org_type=에듀테크기업 & title=팀장 & follow_up_priority=상`
- 근거: 팀장 18명 중 '상' 비중 최다, 대회운영SaaS 연동 관심 집중
- 시퀀스: Day0 연동 중심 개인화 초안 → Day2 미응답시 도입효과 자료 → Day5 기술 도입 미팅 제안
- 우선순위 점수: 90점 (골든타임 민감도 최고)

**P1 — 파트너십 BD 기관공급·공동마케팅 제안서 트랙**
- 세그먼트: `title=사업개발 & interest_feature=파트너십` (12명 전건, 100%)
- 근거: 딜 구조가 기관공급/채널공급/해외·브랜딩 3가지로 분화 → 마이크로세그먼트 대응 필요
- 시퀀스: Day0 인사+제안서 예고 → Day3 수익배분모델 제안서 → Day7 사업개발 미팅 협의
- 우선순위 점수: 80점 (계약 임팩트 최고, 협상 사이클 길어 골든타임 점수는 낮음)

추가 발견 패턴: 원장(20명) 중 65%가 콘텐츠안전필터에 관심 → "안전필터 관심 원장 넛처링" 후보 세그먼트로 별도 확인(이번 라운드 신규 기획에서는 제외, 향후 확장 후보).
