# [05] 키즐링 — 박람회 리드 맞춤 후속 이메일 자동화 (제출)

> 🚀 **한 줄 요약**: 박람회 리드 후속 이메일 8시간 수작업을, 직책별(원장·팀장·사업개발)로 다른 소구점을 반영한 개인화 초안 + 데이터 기반 후속 캠페인 기획안까지 한 번의 Skill 실행으로 만드는 재현 가능한 파이프라인입니다.

- **속도**: 담당자 1인이 리드 50건에 10~15분씩(8시간+) 쓰던 후속 이메일 작성을, `/analyze → /insight → /generate → /review` 4단계(하나의 Skill)로 대체합니다.
- **전환력**: 원장(원생 효과) · 팀장(도입 난이도) · 사업개발(수익 모델)로 소구점·CTA를 다르게 설계해, "누구에게도 안 꽂히는 동일 톤 이메일" 문제를 없앴습니다.
- **실행력**: review_flag(예외 처리) · 이메일 결측 · 완전 중복 리드까지 규칙 기반으로 자동 처리하고, 처리 근거를 모두 `decisions.md`에 기록했습니다.
- **재현성**: 새 박람회 리드 CSV(100건)만 갈아 끼워도 동일한 규칙(직책 매핑·CTA 강도·review_flag 기준)으로 동일 품질 결과가 재현됨을 실제로 검증했습니다.

> 📣 **왜 이 프로젝트는 "마케터의 관점"에서 가장 강한가** — 이 과제의 트랙은 마케팅/PM이고, 승부처는 결국 "리드가 이메일을 열고 답장하게 만드는 카피의 질"입니다. 이 페이지는 직책별 소구점·CTA·호칭까지 실제 상담 맥락에 맞춰 손으로 짠 콘텐츠 설계(§3·§4)를 가장 깊게 다루며, 이는 파이프라인 구조보다 심사에서 더 오래 기억에 남는 부분입니다.

## 1. 문제를 어떻게 재정의했는가

과제 원문의 미션은 "박람회 리드 CSV를 분석해 직책별 소구점이 반영된 후속 이메일 초안을 자동 생성"하는 것이었습니다. `/analyze`로 `data/leads_sample.csv` 50건을 실제로 뜯어보고 나서, 이 과제의 핵심을 다음과 같이 다시 잡았습니다.

> 관건은 "이메일 50통을 빠르게 찍어내는 것"이 아니라, **직책마다 정말 다른 것을 궁금해한다는 사실을 상담 메모에서 읽어내고, 그 근거를 이메일 본문에 그대로 되돌려주는 것**이었습니다. (키즐링의 후속 전환 골든타임은 박람회 후 48시간 — `context/company-info.md`, `context/industry-news.md` 기준)

**데이터를 직접 뜯어보고 확인한 것 (`/analyze` 결과)**
- 직책 분포는 원장 20 / 팀장 18 / 사업개발 12로 어느 한쪽에 쏠리지 않아, 세 세그먼트를 모두 제대로 차별화하지 않으면 리드의 절반 가까이가 소구점이 안 맞는 이메일을 받게 됨.
- 상담 메모(`consultation_note`)를 실제로 읽어보면 원장은 "학부모 만족·원생 재능·유해 콘텐츠 우려"를, 팀장은 "수기 작업 부담·연동 가능 여부"를, 사업개발은 "공급 계약·파트너십 구조"를 각자 구체적으로 언급하고 있었습니다. `company-info.md`가 제시한 직책별 소구점 가설이 실제 리드 발언과 맞아떨어지는 것을 확인한 뒤에야 직책별 CTA를 확정했습니다(`decisions.md` [2026-07-01] 직책별 소구점·CTA 매핑).
- review_flag 대상이 될 상담 메모 결측·모호 리드가 4건(L012·L015·L033·L046), 이메일이 아예 없는 리드가 1건(L009), name·organization·상담 메모까지 완전히 같은 중복 리드가 1쌍(L039=L050) — "50건을 그대로 자동 발송"하면 실제 사고가 날 지점이 이미 원본 데이터 안에 있었습니다.
- title이 "원장"으로 찍혀 있어도 실제로는 초등학교(교장님)·지역아동센터(센터장님)·태권도장(관장님)처럼 호칭 관례가 다른 조직이 3건 섞여 있어, 세그먼트 분류(원장)와 인사말 호칭을 분리해서 처리해야 했습니다.

이 네 가지를 직접 확인하고 나서야 "직책별 소구점 반영"이 템플릿 3개를 만드는 문제가 아니라 **상담 메모를 읽고 예외를 규칙화하는 문제**라는 걸 알았고, 그 판단 기준을 `/insight` 단계에서 review_flag·중복·호칭 규칙으로 구체화했습니다(§4, `decisions.md` 전체).

## 4. 예외 처리 규칙 (review_flag / 결측 / 중복 / 호칭)

| 케이스 | 처리 방식 | 근거 |
|---|---|---|
| 상담 메모 공란·"메모 없음"·모호 | `review_flag = Y`, 본문은 일반화 문구로 대체 | 무리하게 지어내면 신뢰 훼손 위험 — human-in-the-loop이 표준 |

## 5. 재현 가능한 파이프라인 (Standard)

이 프로젝트는 참고 사례처럼 8개 Skill로 쪼개지 않고, **`.claude/skills/SKILL.md` 하나에 4단계를 묶은 진입점**으로 구성했습니다. `/kizzling-lead-followup-email` 한 번 실행(또는 `/analyze → /insight → /generate → /review` 순차 실행)으로 전체 파이프라인이 재현됩니다.

| 단계 | 커맨드 | 역할 |
|---|---|---|
| 0 | 새 leads.csv 교체 | `data/` 폴더에 신규 박람회 리드 CSV 배치 |
| 1 | `/analyze` | 직책·기관유형·우선순위 분포, 이메일/상담메모 결측·모호, 완전 중복 점검 |
| 2 | `/insight` | 직책별 소구점·제품 연결·CTA 매핑, review_flag 기준 수립 |
| 3 | `/generate` | 리드별 이메일 초안 + review_flag 생성, `output/email_drafts.csv` 저장 |
| 4 | `/review` | 직책 차별화·개인화 4요소·본문 품질·예외 처리 자가 점검 |
| 5 | (사람 검토) | review_flag=Y 리드 보강 후 실제 발송은 담당자가 수행 |

별도 `.py` 스크립트나 외부 API 의존 없이, Claude Code 대화 안에서 Skill 실행만으로 전체 체인이 동작합니다.

| review_flag=Y 총계 | 5건 | 10건 (결측·모호 9건 + 이메일결측 1건) |
| 완전 중복 리드 | 1건 (L039=L050) | 0건 |
| 신규 직책값 처리 | 해당 없음 | 교장 6건 / PM 11건 / 주무관 6건 → 매핑 규칙대로 정상 편입 |

→ 결과는 `output/email_drafts_test.csv`로 Basic 제출 대상과 분리 보관하여, **"새 CSV를 넣어도 동일 품질"** 재현성을 증명했습니다.

## 6. [Challenge] 후속 캠페인 기획안 — 리드 패턴 기반

### 6.1 세그먼트 우선순위 판단 기준 (본인 설계)

| 기준 | 가중치 | 설명 |
|---|---|---|
| 세그먼트 규모 × 의향 | 40% | 해당 세그먼트 리드 수 × 우선순위 '상' 비중. 클수록 후속 ROI가 높음 |
| 계약 임팩트 | 40% | 기관공급·파트너십 등 거래 규모. 단가·반복성이 큰 세그먼트 우대 |
| 골든타임 민감도 | 20% | 48시간 내 식기 쉬운가. 데모·미팅 의향이 즉시성 높은 세그먼트 우대 |

### 6.2 캠페인 로드맵 — 담당 부서·리스크 포함 실행형 표

| 우선순위 | 캠페인명 | 세그먼트 정의 | 근거 리드 | 담당 부서 | 핵심 리스크 | 점수 |
|---|---|---|---|---|---|---|
| P0 | 에듀테크 팀장 연동·도입 데모 트랙 | `org_type=에듀테크기업 & title=팀장 & follow_up_priority=상` | L022, L026, L035 | 마케팅팀(초안 발송) → 세일즈엔지니어링팀(연동 자료) → 영업팀(미팅 확정) | 담당(영업팀) 확인이 24시간 넘으면 골든타임(48h) 이탈 | 90점 |
| P1 | 파트너십 BD 기관공급·공동마케팅 제안서 트랙 | `title=사업개발 & interest_feature=파트너십` (12명 전건, 100%) | L039/L050, L044, L049 | 마케팅팀(제안서 예고) → 사업개발팀(제안서·미팅 협의) | 담당(사업개발팀) 확인이 24시간 넘으면 골든타임 이탈, 딜 구조 3종(기관공급/채널/해외) 확인 지연 시 제안서 재작업 위험 | 80점 |

**캠페인 1 — 담당 부서가 명시된 체크리스트형 후속 시퀀스**
- [ ] Day 0 (담당: 마케팅팀) — 팀장용 연동 중심 초안 발송. CTA=기술 도입 미팅, 우선순위 '상'은 48시간 내 미팅 슬롯 제안
- [ ] Day 2 (담당: 세일즈엔지니어링팀) — 미응답 시 도입 효과 수치 자료 + 연동 가이드 전달
- [ ] Day 5 (담당: 영업팀) — 기술 도입 미팅 확정, 운영 공수 절감 수치로 클로징

**캠페인 2 — 담당 부서가 명시된 체크리스트형 후속 시퀀스**
- [ ] Day 0 (담당: 마케팅팀) — 파트너십 인사 + 제안서 예고 발송
- [ ] Day 3 (담당: 사업개발팀) — 수익배분모델 제안서 전달(딜 구조 3종에 맞춰 커스터마이징)
- [ ] Day 7 (담당: 사업개발팀) — 사업개발 미팅 협의, 24시간 내 확인 안 되면 마케팅팀이 리마인드 발송

> 비고: 원장 세그먼트(20명) 중 65%가 콘텐츠안전필터에 관심을 보였으나, 이번 라운드에서는 패턴이 더 뚜렷한 2개 세그먼트(팀장·사업개발)를 우선 선정했습니다. review_flag=Y 리드는 모든 트랙에서 담당 부서 확인 없이 자동 발송하지 않고, 사람 검토 후 합류합니다.
