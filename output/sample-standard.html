# 매 박람회 사용법 (Claude Code 터미널)
# 1. 새 리드 CSV를 data/leads_sample.csv로 교체
/analyze # 직책 분포·이메일/메모 결측·중복 점검
/insight # 원장/팀장/사업개발 소구점 + review_flag 기준
/generate # lead_id·subject·body·review_flag 자동 생성
/review # 직책 차별화·4요소·CTA 자가 점검 후 사람 검토
후속 운영 설계 근거 (본인 설계) 발송은 사람이 최종 검토하므로, AI 초안은 "직책 분기 + 상담 내용 반영 + review_flag"까지만 책임진다. 상담 메모가 비었거나 모호한 리드는 자동 발송 대상에서 빼고 review_flag=Y로 사람에게 넘겨, "기억 못 하는 회사" 인상을 주는 잘못된 자동 발송을 원천 차단했다. 골든타임(48h) 안에 '상' 우선순위부터 처리되도록 정렬한다.
🎯 직책별 소구점·CTA 설계 (본인 기획)

CTA · 기술 도입 미팅

CTA · 파트너십 제안서 송부
📐 우선순위(follow_up_priority)별 CTA 강도 — 직책 × 우선순위 매트릭스
우선순위강도원장팀장사업개발
상 강함 (48h 내) 이번 주 무료 데모 신청 이번 주 기술 도입 미팅 제안서 + 이번 주 미팅
중 보통 (72h 내) 데모 + 사례집 안내 도입 효과 자료 후 미팅 제안 파트너십 제안서 송부

* follow_up_priority는 입력값으로만 사용해 CTA 강도를 조절한다(직접 스코어링 금지). review_flag=Y 리드는 강도와 무관하게 사람 검토 후 발송.
