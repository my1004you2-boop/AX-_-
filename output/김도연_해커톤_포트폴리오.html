EXECUTIVE SUMMARY

한 장 요약

받은 문제 (원문)

박람회에서 수집한 리드 명단 CSV를 분석해, 직책별 소구점이 반영된 개인화 후속 이메일 초안을 자동 생성하라. 원장은 원생효과·학부모만족, 팀장은 도입난이도·공수절감, 사업개발은 수익모델·파트너십을 각각 다르게 소구하고, 상담 메모 결측·모호, 이메일 결측 리드는 예외 처리(review_flag)할 것.

만든 것

리드 CSV만 넣으면 직책별 소구점·CTA가 반영된 개인화 후속 이메일 초안 50건을 자동 생성하고, 결측·모호·중복 리드는 사람이 검토하도록 자동으로 표시하는 파이프라인

문제 이해

기업 원문 문제

박람회 리드 CSV를 넣으면 직책별(원장/팀장/사업개발) 소구점이 반영된 개인화 후속 이메일 초안을 자동 생성하고, 상담 메모·이메일 결측 리드는 예외 처리하라.

내가 재정의한 문제

진짜 문제는 "이메일을 대신 써주는 것"이 아니라, 영업 담당자 1인이 48시간 골든타임 안에 50명 각각의 상담 맥락·직책 니즈를 놓치지 않고 반영하는 것. 데이터가 비어있거나 애매한데 억지로 채우면 오히려 신뢰를 잃으므로, 예외를 정확히 골라내는 것이 자동화의 핵심 난이도.

문제의 크기 (Before)

리드 1명당 후속 이메일 작성 10~15분 → 50명이면 8시간 이상, 담당자 1인 감당 불가. 후속이 늦어지면 48시간 골든타임을 놓쳐 경쟁사에 리드를 뺏김.

스코프 — 푼 것 / 안 푼 것

O 직책별 소구점·CTA 매핑, 개인화 4요소 초안 50건, 결측·모호·중복 예외처리, 100건 새 CSV 재현성 검증, 캠페인 기획안 2건 / X 실제 발송(SMTP), CRM 연동, 리드 스코어링 직접 산출, 다국어, 대시보드

APPROACH & DECISION

해결 접근 & 의사결정

접근 가설 / 핵심 흐름

리드 CSV 확인 → 직책·데이터품질 분석 → 직책별 소구점·CTA 규칙 설계 → 리드별 초안 자동 생성 → 자가 점검. 데이터가 비어있거나 애매한 리드는 사람이 검토하도록 별도로 표시.

버린 대안 & 이유

전 직책 공통 CTA("상담 신청")로 통일 — 원장(단독결정)·팀장(실무검토)·사업개발(계약구조설계)의 결정 구조가 달라 효과가 떨어질 것으로 판단해 폐기. 우선순위 낮은 리드까지 강하게 미팅을 요청하는 안도 이탈 위험으로 폐기.

★ 막혔던 지점 & 돌파 방법

직책이 "원장"으로 똑같이 적혀 있어도 실제로는 초등학교 교장·지역아동센터 센터장·태권도장 관장이 섞여 있어, 그대로 "원장님"이라 부르면 어색했다. → 소구점·CTA는 원장 그룹 그대로 두고 인사말 호칭만 교장님/센터장님/관장님으로 바꾸는 규칙을 추가. 이후 100건 재현성 테스트에 등장한 신규 직책값(교장·PM·주무관)도 같은 원리로 매핑 규칙을 세워 규칙이 깨지지 않게 했다.

AI vs 내 판단

AI에 맡긴 것: 50건 개인화 이메일 문구 초안 생성, 결측·모호 데이터 자동 탐지. 내가 직접 결정한 것: 직책별 소구점·CTA 매핑 기준, review_flag 판단 기준, 우선순위×직책 CTA 강도 매트릭스, 캠페인 세그먼트 선정 가중치(규모×의향 40%·계약임팩트 40%·골든타임 20%).

입력

처리

직책별 소구점 규칙(원장=원생효과·팀장=도입난이도·사업개발=수익모델)에 맞춰 인사말·상담내용 언급·제품 연결·CTA를 조합. 예) "영어스피치 대회를 수기로 진행해 힘들다"는 원장님 상담 메모 → 대회 자동화 소개 + 무료 데모 CTA로 연결. 상담 메모가 비었거나 애매하면 review_flag=Y, 이메일이 없으면 발송 대신 내부메모로 전환.

결과

제목·본문(250~400자)·review_flag가 채워진 이메일 초안 50건(email_drafts.csv). 예외 5건은 사람이 보강 후 발송하도록 별도 표시.

핵심: 담당자는 기술을 모른다. "어떻게 만들었나"가 아니라 "무슨 일을 대신해 주는가"로 설명. 깊은 기술 디테일은 별첨으로.
